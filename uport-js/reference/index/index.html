Credentials

The Credentials class allows you to easily create the signed payloads used in uPort inlcuding credentials and signed mobile app requests (ex. selective disclosure requests for private data). It also provides signature verification over signed payloads and allows you to send push notifications to users.

uport-js/JWT

  • uport-js/JWT

    • .createJWT([config], payload) ⇒ Promise.<Object, Error>
    • .verifyJWT([config], jwt, callbackUrl) ⇒ Promise.<Object, Error>

uport-js/JWT.createJWT([config], payload) ⇒ Promise.<Object, Error>

Creates a signed JWT given an address which becomes the issuer, a signer, and a payload for which the signature is over.

Kind: static method of uport-js/JWT
Returns: Promise.<Object, Error> - a promise which resolves with a signed JSON Web Token or rejects with an error

ParamTypeDescription
[ config ] Object an unsigned credential object
config.address String address, typically the uPort address of the signer which becomes the issuer
config.signer SimpleSigner a signer, reference our SimpleSigner.js
payload Object payload object

Example 

const signer = SimpleSigner(process.env.PRIVATE_KEY)
 createJWT({address: '5A8bRWU3F7j3REx3vkJ...', signer}, {key1: 'value', key2: ..., ... }).then(jwt => {
     ...
 })

uport-js/JWT.verifyJWT([config], jwt, callbackUrl) ⇒ Promise.<Object, Error>

Verifies given JWT. Registry is used to resolve uPort address to public key for verification. If the JWT is valid, the promise returns an object including the JWT, the payload of the JWT, and the profile of the issuer of the JWT.

Kind: static method of uport-js/JWT
Returns: Promise.<Object, Error> - a promise which resolves with a response object or rejects with an error

ParamTypeDescription
[ config ] Object an unsigned credential object
config.address String address, typically the uPort address of the signer which becomes the issuer
config.registry UportLite a uPort registry, reference our uport-lite library
jwt String a JSON Web Token to verify
callbackUrl String callback url in JWT

Example 

const registry =  new UportLite()
 verifyJWT({registry, address: '5A8bRWU3F7j3REx3vkJ...'}, 'eyJ0eXAiOiJKV1QiLCJhbGciOiJFUzI1NksifQ.eyJyZXF1Z....').then(obj => {
     const payload = obj.payload
     const profile = obj.profile
     const jwt = obj.jwt
     ...
 })

Credentials

The Credentials class allows you to easily create the signed payloads used in uPort inlcuding credentials and signed mobile app requests (ex. selective disclosure requests for private data). It also provides signature verification over signed payloads and allows you to send push notifications to users.

Kind: global class

  • Credentials

    • new Credentials([settings])
    • .createRequest([params]) ⇒ Promise.<Object, Error>
    • .createVerificationRequest(unsignedClaim, sub) ⇒ Promise.<Object, Error>
    • .receive(token, [callbackUrl]) ⇒ Promise.<Object, Error>
    • .push(token, payload, pubEncKey) ⇒ Promise.<Object, Error>
    • .attest([credential]) ⇒ Promise.<Object, Error>
    • .lookup(address) ⇒ Promise.<Object, Error>

new Credentials([settings])

Instantiates a new uPort Credentials object

ParamTypeDescription
[ settings ] Object setttings
settings.networks Object networks config object, ie. { '0x94365e3b': { rpcUrl: ' https://private.chain/rpc' , address: '0x0101.... }}
settings.registry UportLite a registry object from UportLite
settings.signer SimpleSigner a signer object, see SimpleSigner.js
settings.address Address your uPort address (may be the address of your application's uPort identity)

credentials.createRequest([params]) ⇒ Promise.<Object, Error>

Creates a signed request token (JWT) given a request params object.

Kind: instance method of Credentials
Returns: Promise.<Object, Error> - a promise which resolves with a signed JSON Web Token or rejects with an error

ParamTypeDefaultDescription
[ params ] Object {} request params object
params.requested Array an array of attributes for which you are requesting credentials to be shared for
params.verified Array an array of attributes for which you are requesting verified credentials to be shared for
params.notifications Boolean boolean if you want to request the ability to send push notifications
params.callbackUrl String the url which you want to receive the response of this request
params.network_id String network id of Ethereum chain of identity eg. 0x4 for rinkeby
params.accountType String Ethereum account type: "general", "segregated", "keypair", "devicekey" or "none"

Example 

const req = { requested: ['name', 'country'],
               callbackUrl: 'https://myserver.com',
               notifications: true }
 credentials.createRequest(req).then(jwt => {
     ...
 })


 requested: ['name','phone','identity_no'],
    callbackUrl: 'https://....' // URL to send the response of the request to
    notifications: true

credentials.createVerificationRequest(unsignedClaim, sub) ⇒ Promise.<Object, Error>

Creates a signed request for the user to attest a list of claims.

Kind: instance method of Credentials
Returns: Promise.<Object, Error> - a promise which resolves with a signed JSON Web Token or rejects with an error

ParamTypeDescription
unsignedClaim Object an object that is an unsigned claim which you want the user to attest
sub String the DID of the identity you want to sign the attestation

Example 

const unsignedClaim = {
   claim: {
     "Citizen of city X": {
       "Allowed to vote": true,
       "Document": "QmZZBBKPS2NWc6PMZbUk9zUHCo1SHKzQPPX4ndfwaYzmPW"
     }
   },
   sub: "2oTvBxSGseWFqhstsEHgmCBi762FbcigK5u"
 }
 credentials.createVerificationRequest(unsignedClaim).then(jwt => {
   ...
 })

credentials.push(token, payload, pubEncKey) ⇒ Promise.<Object, Error>

Send a push notification to a user, consumes a token which allows you to send push notifications and a url/uri request you want to send to the user.

Kind: instance method of Credentials
Returns: Promise.<Object, Error> - a promise which resolves with successful status or rejects with an error

ParamTypeDescription
token String a push notification token (get a pn token by requesting push permissions in a request)
payload Object push notification payload
payload.url String a uport request url
payload.message String a message to display to the user
pubEncKey String the public encryption key of the receiver, encoded as a base64 string

credentials.attest([credential]) ⇒ Promise.<Object, Error>

Create a credential (a signed JSON Web Token)

Kind: instance method of Credentials
Returns: Promise.<Object, Error> - a promise which resolves with a credential (JWT) or rejects with an error

ParamTypeDescription
[ credential ] Object a unsigned credential object
credential.sub String subject of credential (a uPort address)
credential.claim String claim about subject single key value or key mapping to object with multiple values (ie { address: {street: ..., zip: ..., country: ...}})
credential.exp String time at which this claim expires and is no longer valid (seconds since epoch)

Example 

credentials.attest({
  sub: '5A8bRWU3F7j3REx3vkJ...', // uPort address of user, likely a MNID
  exp: <future timestamp>,
  claim: { name: 'John Smith' }
 }).then( credential => {
  ...
 })
